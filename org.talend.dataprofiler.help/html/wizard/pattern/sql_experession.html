SQL Expressions:

In the first page of the wizard, set the name of the pattern so that you can find it easily.

In the second page, set the sql pattern to be used in analyses.
The pattern must be surrounded with single quotes "'".
Once the pattern is created, you can drag & drop it onto a column in the Multiple Column Analysis editor.

Note:
SQL patterns are different from Regular patterns.
For example, if you search strings starting with 'S', the regular pattern is 'S%', not 'S.*'.
SQL pattern are more easily supported by the databases:
they appear in the « LIKE » clause of an SQL statement.
Queries with SQL patterns can be faster than queries with regular expressions.
But they are usually far less powerful.
Please refer to the documentation of your Database Management System for more details.
